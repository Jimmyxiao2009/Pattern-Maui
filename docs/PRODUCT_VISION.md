# Pattern 产品愿景：贾维斯式桌面伴随体

> 版本 0.4 · 2026-07-14  
> 一句话：**Pattern = Claude Code（能编码） + Codex（能交付工作） + 酒馆式角色扮演（有人格陪伴）**，统一成常驻 OS 的「贾维斯」。

---

## 1. 产品定义

Pattern 不是单一聊天机器人，也不是纯 IDE 插件。它是：

| 角色 | 含义 |
|------|------|
| **编码伙伴** | 绑定本地项目，读仓、改仓、看 diff、跑命令（对标 Claude Code / Cursor Agent） |
| **工作代理** | 长任务、Computer Use、审批、产物与进度可见（对标 Codex Work） |
| **人格陪伴** | 用户定义的性格、长期记忆、主动开口、托盘常驻（对标酒馆/角色卡 + OS 居民） |

用户心智模型：**贾维斯**——记得你、会找你、能动手、能写代码、干活过程可审。

---

## 2. 三支柱体验标准

### 2.1 编码（Coding）

**优秀标准**

- 打开项目 = 真实工作目录进入模型上下文  
- 文件树可浏览、可预览、可 @ 引用  
- 改动可 diff / worktree 隔离 / 审查后合并  
- 对话内可见工具与命令输出  
- 无能力时明确「只分析」，禁止假完成  

**当前水位（诚实）**

- 有三栏壳与 project/global 对话  
- 项目上下文未真正注入模型  
- 文件树浅、不可预览  
- worktree/diff 与项目页脱节  

### 2.2 工作（Work）

**优秀标准**

- 派活 → 过程可见 → 可停可接管 → 结果可回看  
- 对话时间线含：思考 / 工具 / 截屏 / 审批 / 完成  
- 多任务不串号；审查窗绑定 taskId  
- 计划任务有 run history 与失败策略  

**当前水位**

- 有任务、审查窗、T0–T3、急停  
- 主对话看不到执行步骤  
- 无 Stop/Retry；执行与聊天脱节  

### 2.3 陪伴（Companionship）

**优秀标准**

- 稳定人格（角色卡）与双槽（陪伴/执行）  
- 长期记忆可浏览、可编辑、可溯源  
- 主动性有分寸：忙时轻插，空闲可拉起  
- 快捷窗共享短历史，像同一个人  
- 托盘常驻，关窗不退出  

**当前水位**

- 人格 / 记忆 / 主动 / 托盘已具备雏形  
- 主动消息强制新会话且可能打断  
- 记忆不可编辑；快捷窗 history 为空

## 3. 与竞品的关系（不抢赛道）

| 产品 | 学什么 | 不学什么 |
|------|--------|----------|
| Claude Code | 权限模式、并行 session、diff/终端一体 | 不做纯终端 CLI 优先 |
| Codex | 项目/任务组织、Computer Use 审批、产物 | 不做封闭 OpenAI 账号墙 |
| 酒馆 / 角色卡 | 人格定义、说话方式、关系感 | 不做纯文学 RP 沙盒 |
| Cursor | 仓库上下文与改码闭环 | 不做完整 IDE |

Pattern 的差异化：**本地优先的人格 OS Agent**——编码与工作是硬能力，陪伴是关系层，三者同一进程、同一记忆、同一审计。

---

## 4. 体验原则

1. **禁止假完成**：UI 显示「能做」就必须接到真能力，否则标明限制。  
2. **过程即信任**：执行时间线比最终一句话更重要。  
3. **上下文连续**：全局 / 项目 / 快捷窗 / 任务共享人格与可解释的上下文边界。  
4. **陪伴不抢戏**：主动性默认可配置；编码/工作时不硬打断。  
5. **用户可纠错**：记忆、任务、对话失败均可编辑/重试，而不是只能删。  

---

## 5. 成功判据（用户故事）

1. **编码**：打开仓库 → 问「这个模块怎么改」→ 模型带着路径与文件回答 → 改动能看 diff。  
2. **工作**：说「帮我整理下载文件夹」→ 对话里看到步骤 → 可暂停/接管 → 审查窗与任务一致。  
3. **陪伴**：夜里主动提醒休息 → 可选轻插当前会话 → 记得你上周说过的偏好 → 快捷窗接着聊。  

三块都通过，才算「贾维斯」而不是「三个半成品拼盘」。
