# Pattern 体验缺口与路线图

> 与 `PRODUCT_VISION.md` 配套。按 **编码 / 工作 / 陪伴** 记录用户可见缺陷与关闭路径。

---

## 1. 编码缺口

| 级别 | 症状 | 关闭标准 |
|------|------|----------|
| P0 | 项目聊天不注入 workspace | `chat.send` 带 `workspace`/`projectName`；system prompt 含路径 |
| P0 | 文件树不可打开 | 点击文件可预览文本；显示路径 |
| P0 | 文件树 depth=2 过浅 | 目录展开可懒加载；depth 提升或按需 list |
| P1 | 建项目手填路径 | 文件夹选择器（Tauri dialog 或输入+校验） |
| P1 | 无项目内 diff | 项目页「查看 diff」调用 `workspace.diff` |
| P1 | 工作流与项目脱节 | 运行工作流默认填当前项目 path |
| P2 | 附件 64KB / 无 @ 文件 | @ 文件树节点附加到 draft；提高文本上限策略 |
| P2 | skills 硬编码 | 可安装 skill 清单（后续） |

---

## 2. 工作缺口

| 级别 | 症状 | 关闭标准 |
|------|------|----------|
| P0 | 对话无工具/步骤时间线 | 消息可挂 `events[]`；任务更新映射进当前对话 |
| P0 | 无法 Stop / Retry | 前端 abort 标记 + 停止生成；失败气泡可重试 |
| P1 | 审查窗绑错任务 | URL/`taskId` 参数或最近创建任务优先 |
| P1 | 无 Markdown | 助手消息渲染基础 markdown（代码块/列表） |
| P1 | 转交执行与对话脱节 | 转交后在气泡内显示 task 卡片/进度摘要 |
| P2 | MCP 无试调 | MCP 页「调用」调试入口 |

---

## 3. 陪伴缺口

| 级别 | 症状 | 关闭标准 |
|------|------|----------|
| P1 | 主动消息强制新会话+抢焦点 | 设置：`new_chat` / `inline`；默认可配置 |
| P1 | 快捷窗 history=[] | 读取最近 global 对话短 history |
| P1 | 记忆不可编辑 | `memory.update` + 编辑器 |
| P1 | 错误覆盖气泡 | 错误独立展示 + 重试按钮 |
| P2 | 前台感知占位 | 接 OS 前台窗口 API（后续） |
| P2 | OOBE 后无导览 | 首启 checklist（后续） |

---

## 4. 阶段路线

### 阶段 A — 消灭假完成（当前冲刺）
- 项目上下文真注入  
- 文件预览 + 懒加载  
- 对话事件时间线 + Stop/Retry  
- Markdown 基础渲染  
- 审查窗绑定 task  
- 主动消息策略 + 快捷窗历史  
- 记忆编辑  

### 阶段 B — 交付闭环
- 项目页 diff / worktree 一键  
- 工作流绑定当前项目  
- 任务 run history 强化  
- 对话内 task 卡片  

### 阶段 C — 贾维斯打磨
- [x] 主动性分寸（忙/闲静默）  
- [x] 记忆溯源跳转  
- [x] 前台窗口标题感知（忙闲启发式）  
- [x] Skills 安装与 MCP 试调  

---

## 5. 验收脚本建议

- UI：项目页发消息后 system/status 显示「工作区：path」  
- UI：点击 README 可预览  
- UI：发送后可点停止；失败可重试  
- UI：主动消息在 `inline` 策略下不强制新会话  
- `pnpm check` + `pnpm test:ui`
